@mixin generate-col-span()

Generates a grid column span mixin

Parameters

$max: (Number)

The maximum number of columns to span

Examples

scss 
@include generate-col-span(4);
css compiled 
.col-span-1 {
  grid-column: span 1;
}

.col-span-2 {
  grid-column: span 2;
}

.col-span-3 {
  grid-column: span 3;
}

.col-span-4 {
  grid-column: span 4;
}
scss 
.block {
  @include generate-col-span(3);
}
css compiled 
.block .col-span-1 {
  grid-column: span 1;
}
.block .col-span-2 {
  grid-column: span 2;
}
.block .col-span-3 {
  grid-column: span 3;
}

@mixin content-grid()

This mixin is used to create a grid layout with a responsive content area with a maximum width. It also has the ability to create full-width sections. It is mainly used in the header, footer, and main sections of the page.

Example

scss 
main {
  @include content-grid;
}
css compiled 
main {
  display: grid;
  grid-template-columns: [full-width-start] minmax(var(--padding-inline), 1fr) [content-start] min(100% - var(--padding-inline) * 2, var(--content-max-width)) [content-end] minmax(var(--padding-inline), 1fr) [full-width-end];
  transition: padding var(--transition-duration-fast);
}
main > :not(.full-width) {
  grid-column: content;
}
main > .full-width > :not(.full-width) {
  grid-column: content;
}
main > .full-width {
  display: grid;
  grid-column: full-width;
  grid-template-columns: inherit;
  padding-block: var(--layout-row-spacing);
}
main > .full-width:last-of-type {
  padding-bottom: 0;
}

A mixin for generating media queries based on breakpoints, units, and custom media features. It supports breakpoints defined in various units (em, rem, px) or as named keys from a breakpoints map.The mixin allows configuration of min-width, max-width, min-height, or max-height queries,and it optionally supports additional conditions such as orientation.

Parameters & Return

$value: (Number|String)

The breakpoint value or key from the bp.$breakpoints-map.

  • Can be a numeric value (e.g., 768px) or a named key from the breakpoints map (e.g., 'medium').

$type: min (String)

The type of media query.

  • Options: 'min' or 'max'.
  • Defaults to 'min'.

$media-feature: width (String)

The feature to target.

  • Options: 'width' or 'height'.
  • Defaults to 'width'.

$orientation: null (String)

An optional orientation condition.

  • Options: 'portrait' or 'landscape'.
  • Defaults to null (no orientation condition).

@content (code block)

- The CSS rules to apply within the media query.

@error

  • An error if an invalid or missing breakpoint value is provided.

Example

scss 
// Using a key from the breakpoints map (default: min-width)
@include mq('md') {
  .example { color: blue; }
}

// Using an explicit max-width query
@include mq('md', 'max') {
  .example { color: red; }
}

// Using a custom breakpoint value
@include mq(768px) {
  .example { font-size: 1.2rem; }
}

// Adding an orientation condition
@include mq('md', $orientation: 'landscape') {
  .example { color: green; }
}

// Targeting height instead of width
@include mq(1024px, $media-feature: 'height') {
  .example { background-color: yellow; }
}
css compiled 
@media screen and (min-width: 64em) {
  .example {
    color: blue;
  }
}
@media screen and (max-width: 64em) {
  .example {
    color: red;
  }
}
@media screen and (min-width: 768px) {
  .example {
    font-size: 1.2rem;
  }
}
@media screen and (min-width: 64em) and (orientation: landscape) {
  .example {
    color: green;
  }
}
@media screen and (min-height: 1024px) {
  .example {
    background-color: yellow;
  }
}
